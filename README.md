# 足球预测实验室：每日预测与验真

这是原“2026 世界杯 Elo 蒙特卡洛预测器”的持续验证版本。系统不再局限于杯赛，而是每天读取中国体彩网竞彩足球公开赛程，只发布北京时间未来两日、尚未开赛的足球预测，并在赛后自动验真。

## 输出内容

每场比赛固定输出五项：

1. 胜平负
2. 让球胜平负（使用体彩网当场官方让球线）
3. 比分
4. 总进球数（0—6、7+）
5. 半全场胜平负

首页同时展示累计样本数、各玩法准确率、五项全中率、历史逐场预测与真实赛果。

## 模型闭环

- 数据：体彩网竞彩足球公开赛程、胜平负/让球赔率、半场与全场赛果，以及每场“赛事前瞻”中的近期战况、跨年份交锋、射手贡献和伤停；可选接入经过授权的场外赔率与赛前情报 JSON 源。
- 客观层：首次回填近两年官方赛果，学习球队近期进失球、滚动 Elo、联赛主客场进球、平局率、半场平局率和真实比分分布；每天再同步单场详情页补足体彩赛果列表未覆盖的比赛。
- 盘口层：官方与场外赔率统一去水后融合；场外盘最多占市场共识的 30%。若体彩网因实力悬殊只开让球胜平负，模型会提高让球盘拟合权重并反推真实实力差，不把它误判为数据缺失。
- 近期层：赛事前瞻近 10 场胜平负以最高 12% 并入方向判断，场均进失球以最高 24% 修正球队进球基线，避免短样本覆盖赔率和长期实力。
- 交锋层：优先使用赛事前瞻的跨年份交锋，最近 8 次按 540 天时间常数衰减，方向权重最高 10%。页面显示“无匹配”仅代表该详情源未覆盖，不声称两队从未交手。
- 模拟层：每场固定种子运行 20,000 次带比赛节奏波动的蒙特卡洛，保留 0:0/1:1 等低比分，也保留 2:3、3:2、1:4 等开放型长尾。
- 分玩法决策：胜平负、让球胜平负和半全场分别按各自校准后的边际概率或联合概率取第一名；代表比分只是一条独立的高概率比赛路径，不再反向覆盖让球或半全场主选。这样避免常见的一球差比分把整批预测机械推成“让球平”。
- 软联合选分：比分概率、联赛历史比分、xG 距离和让球支持度共同评分；让球前两项差小于 5 个百分点标为“临界”，小于 10 个百分点标为“偏弱”。随机流只绑定比赛 ID，不随模型补丁版本漂移。
- 平局建模：市场平赔、联赛平局率和半场平局率分别进入全场与半场判断，页面同时展示平局概率及半场平候选。
- 情报层：赛事前瞻伤停与射手贡献自动关联，按位置、首发比例、进球占比和助攻占比计算受限 xG 修正；页面注明部分数据来自第三方，因此自动置信度为 74%。`data/adjustments.json` 仍可录入已经核验的教练变动、轮换、天气和动机；主客 xG 总修正各自限制在 ±0.45。
- 学习：完赛后根据赛前已冻结样本的 Brier 分数、进球残差和主场残差，收缩更新市场权重、总进球尺度和主场修正。

预测写入 `data/history.json` 后即冻结。模型升级不会回算或改写旧预测，因此历史准确率是可审计的真实前瞻成绩，不是赛后回放拟合成绩。

## 本地运行

需要 Node.js 20 或更高版本，无第三方依赖。

```bash
npm test
npm run update
npm run serve
```

浏览器打开 `http://127.0.0.1:4173`。Windows 也可以直接双击 `启动本地预览.bat`；如需先联网更新，双击 `立即联网更新.bat`。

## 自动更新

由于体彩网公开接口会限制部分境外机房，数据更新应在能够正常访问接口的国内机器上每 30 分钟运行一次 `scripts/auto-update.ps1`：

1. 拉取未来赛程与近 45 天赛果；
2. 结算已冻结预测；
3. 更新 Elo 和校准参数；
4. 仅为未开赛且未发布过的比赛生成预测；
5. 提交 `data/*.json`，GitHub Pages 随之更新。

脚本会先同步 `main`、运行测试、生成数据、再次测试，然后只在 `data` 确有变化时提交并推送。它使用文件锁防止重叠运行，工作区存在未提交改动时会安全退出。

`.github/workflows/update-predictions.yml` 不再从 GitHub 海外运行器抓取体彩网，而是每两小时检查一次公开数据是否在四小时内更新。数据过期时工作流会明确失败并触发 GitHub 通知，避免“绿色假成功”。

若仓库启用了分支式 GitHub Pages，网页入口仍是：

`https://xkxjj.github.io/daily-football-predictor/`

## 可选的联网场外数据

默认情况下系统只自动使用体彩网和两年历史赛果，不会抓取来源不明的赔率或传闻。若有已授权的数据服务，可在 GitHub 仓库中配置：

- Actions Variable：`FOOTBALL_CONTEXT_FEED_URL`，指向 HTTPS JSON 地址；
- Actions Secret：`FOOTBALL_CONTEXT_FEED_TOKEN`，若数据源需要 Bearer Token 才填写。

数据以体彩网 `matchId` 或“主队|客队”为键，支持 `externalMarket`、`teamNews`、`coachNews`、`expiresAt` 等字段。完整示例见 `data/adjustments.json`。联网信息与本地复核信息会自动合并，本地值优先；数据源不可用时仍继续使用官方数据生成预测。

## 重要说明

准确率随玩法难度差异很大，尤其“精确比分”天然是低命中任务；界面不会把不同玩法混成一个看似很高的数字。任何预测都不能保证正确或盈利，本项目只用于模型研究、公开验证和复盘，不构成投注建议。
