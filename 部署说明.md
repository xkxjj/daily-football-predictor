# 部署说明

## GitHub Pages

1. 将本目录提交并推送到 GitHub 仓库。
2. 仓库 `Settings → Pages` 选择从主分支根目录发布。
3. 在 `Actions` 页面手动运行一次 `Update daily football predictions`。
4. 确认仓库 `Settings → Actions → General → Workflow permissions` 允许工作流写入仓库。

此后工作流每 30 分钟拉取体彩网数据并提交 `data` 目录。Pages 会在数据提交后自动刷新。

## 其他静态托管

Cloudflare Pages、Netlify、Vercel 均可直接发布仓库根目录，但自动更新仍由 GitHub Actions 写回仓库完成。项目不需要构建命令；输出目录就是仓库根目录。

## 本地

先运行 `立即联网更新.bat`，再运行 `启动本地预览.bat`。不要直接双击 `index.html`，浏览器对本地 JSON 的安全限制会阻止数据加载。
